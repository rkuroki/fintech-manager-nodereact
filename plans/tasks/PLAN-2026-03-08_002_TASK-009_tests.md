# TASK-009: Tests

## Goal
Write comprehensive backend tests for the Gmail domain. Maintain existing coverage thresholds.

## Changes

### `packages/backend/src/domains/gmail/gmail.test.ts` (NEW)

**Test categories:**

#### 1. Gmail Not Configured (graceful degradation)
- `GET /api/gmail/status` → 501 when `GOOGLE_CLIENT_ID` not set
- `GET /api/gmail/auth` → 501 when not configured
- `POST /api/gmail/sync/:id` → 501 when not configured
- `GET /api/gmail/timeline/:id` still works (returns manual records only)

#### 2. Authentication & Permissions
- All routes return 401 without JWT
- `GET /api/gmail/status` returns 403 for users without `GMAIL_CONNECT`
- `POST /api/gmail/sync/:id` returns 403 for users without `GMAIL_SYNC`
- `GET /api/gmail/timeline/:id` allowed for users with `COMMUNICATIONS_READ`
- Admin bypasses all permission checks

#### 3. Connection Status
- Returns `{ connected: false }` for new user (no tokens)
- Returns `{ connected: true, email: ... }` when tokens exist

#### 4. OAuth Flow (with mocked Google API)
- `GET /api/gmail/auth` returns valid Google consent URL with correct scopes
- `GET /api/gmail/callback` exchanges code for tokens (mock Google token endpoint)
- `GET /api/gmail/callback` stores encrypted tokens in DB
- `GET /api/gmail/callback` redirects to frontend

#### 5. Disconnect
- `DELETE /api/gmail/disconnect` removes tokens from DB
- `DELETE /api/gmail/disconnect` deletes cached messages for user
- `DELETE /api/gmail/disconnect` returns 204

#### 6. Email Sync (with mocked Gmail API)
- `POST /api/gmail/sync/:id` fetches and caches emails
- Sync returns count of synced messages
- Incremental sync only fetches new messages
- Sync handles customer with no email → returns `{ synced: 0 }`
- Sync handles expired token → refreshes before fetching

#### 7. Unified Timeline
- `GET /api/gmail/timeline/:id` returns merged manual + Gmail entries
- Entries sorted by timestamp descending
- Manual entries have `type: 'manual'`
- Gmail entries have `type: 'gmail'`
- Pagination works correctly on merged results

**Mocking strategy:**
- Use `vi.mock` to mock `fetch` for Google API calls (token exchange, Gmail message list, message get)
- Use in-memory SQLite for DB (same pattern as existing tests)
- Pre-populate test data: create test user, customer, manual communications, and seed gmail_messages directly
